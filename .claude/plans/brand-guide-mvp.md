# Brand Guide вЂ” Siza Brand + Reusable Template Static Site

## Context

The `Forge-Space/brand-guide` repo is **empty**. Goal: build a static brand guide site that serves two purposes:
1. **Siza's canonical brand reference** вЂ” logo, colors, typography, tokens, usage rules
2. **Reusable template** вЂ” other projects/users can fork and populate with their own brand identity (generated by branding-mcp)

### What Already Exists
- **branding-mcp v0.5.0**: 9 MCP tools generating complete `BrandIdentity` objects with 6 export formats (CSS, JSON, Tailwind, React, Sass, Figma). Produces logos (4 SVG variants), favicons (4 sizes), OG images (3 templates), gradients, shadows, motion, borders.
- **Siza webapp**: Uses `#7c3aed` purple, Inter/Outfit/JetBrains Mono fonts, dark theme with 5 surface grays. No systematic design tokens вЂ” colors hardcoded across ~15 files.
- **Strategic positioning**: "The Open Full-Stack AI Workspace" / "Generate. Integrate. Ship."
- **Domain**: `forgespace.co` with Cloudflare DNS (zone `63f7ebf9c355679fdfad35abbe632986`)

## Phase 1: Project Scaffold + Brand Generation (~1h)

**Goal:** Clone repo, scaffold Astro project, generate Siza's brand identity, commit foundation.

### Steps

1. Clone the empty repo:
   ```bash
   cd ~/Desenvolvimento/forge-space
   git clone git@github.com:Forge-Space/brand-guide.git
   cd brand-guide
   ```

2. Initialize Astro project with Tailwind:
   ```bash
   npm create astro@latest . -- --template minimal --no-install --typescript strict
   npx astro add tailwind --yes
   npm install
   ```

3. Generate Siza brand identity by calling branding-mcp's `generate_design_system` tool with these parameters:
   - `brandName`: "Siza"
   - `industry`: "technology"
   - `style`: "tech" (maps to current purple + monospace aesthetic)
   - `baseColor`: "#7c3aed" (preserve existing primary)
   - `harmony`: "analogous" (purple в†’ blue в†’ violet range)
   - `theme`: "both" (dark-first but support light)
   - `headingCategory`: "sans-serif" (Outfit-like)
   - `bodyCategory`: "sans-serif" (Inter-like)
   - `scaleRatio`: "major-third" (1.25 вЂ” clean progression)
   - `exportFormats`: ["css", "json", "tailwind"]

4. Save outputs:
   - `brand/identity.json` вЂ” full BrandIdentity object
   - `src/styles/tokens.css` вЂ” CSS custom properties
   - `public/downloads/tokens.json` вЂ” W3C design tokens
   - `tailwind.config.ts` вЂ” extend with generated preset
   - `public/logos/` вЂ” 4 SVG variants (wordmark, monogram, abstract, icon)
   - `public/favicons/` вЂ” 4 sizes
   - `public/og/` вЂ” 3 OG templates

5. Create `CLAUDE.md` with project conventions

6. Create initial `README.md`

### Verification
- [ ] `npm run dev` starts Astro dev server
- [ ] `brand/identity.json` contains valid BrandIdentity
- [ ] `src/styles/tokens.css` has CSS variables
- [ ] Logo SVGs render correctly in browser

### Anti-Patterns
- Don't manually pick colors вЂ” use branding-mcp with `baseColor: "#7c3aed"` to maintain consistency
- Don't install React вЂ” Astro's native components + islands are sufficient for a brand guide

## Key Gotchas

- **branding-mcp `generate_design_system` returns strings, not files** вЂ” must parse exports object and write to disk manually
- **Logo SVGs are inline strings** вЂ” write to .svg files, ensure viewBox is preserved
- **Astro + Tailwind**: Use `@astrojs/tailwind` integration, not manual PostCSS setup
- **Cloudflare Pages build**: Node 22 must be set via `NODE_VERSION` env var in Pages settings
- **CLAUDE.md gitignored ecosystem-wide**: Use `git add -f CLAUDE.md` if committing
- **`identity.json` is the contract**: Everything derives from it вЂ” never hardcode token values in components
- **branding-mcp fonts are names only**: Google Fonts must be loaded separately (link tag or `@fontsource`)
- **WCAG contrast data**: `contrast` field uses color-pair keys like `"primary-on-white"` вЂ” parse carefully
- **Astro islands**: Only use `client:load` for truly interactive components (contrast checker). Everything else is static.
